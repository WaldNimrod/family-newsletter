## 1. סקירה כללית

המערכת מייצרת **ניוזלטר משפחתי יומי** המותאם אישית לתחומי העניין של כל בן משפחה. הניוזלטר נבנה כדף HTML שמועלה לשרת קבוע, וקישור עם כותרת נשלח לכל בן משפחה בוואטסאפ.

### עקרונות מנחים

- **פרסונליזציה**: כל בן משפחה מגדיר תחומי עניין וערוצי מדיה משלו
- **תמהיל**: שילוב חכם בין תכנים מכל בני המשפחה + תוכן שהועלה ידנית
- **דו-שפתיות**: תכנים בעברית ובאנגלית, עם תיוג שפה
- **משוב מתמשך**: סקר מהיר בוואטסאפ לשיפור התכנים
- **אוטומציה מלאה**: מ-Curation ועד שליחה - בלי מגע יד אדם

## 3. מקורות מידע ונוהל יומי

### סוגי מקורות

| סוג | דוגמאות | שיטת איסוף |
|-----|---------|------------|
| RSS Feeds | בלוגים, אתרי חדשות | `feedparser` |
| אתרי חדשות | Ynet, Walla, TechCrunch | Web scraping / RSS |
| YouTube | ערוצים מועדפים | YouTube Data API |
| Twitter/X | חשבונות מעניינים | Twitter API |
| פודקאסטים | פרקים חדשים | Podcast RSS |
| תוכן משפחתי | תמונות, סיפורים, עדכונים | WhatsApp upload |

### תוכן שמועלה ע"י בני המשפחה

בני המשפחה יכולים לשלוח תוכן ישירות לבוט בוואטסאפ:
- **תמונה + כיתוב** → מופיעה במדור "מהמשפחה"
- **קישור + תגובה** → נכנס לתמהיל התכנים
- **טקסט חופשי** → סיפור/עדכון אישי

### נוהל יומי (Pipeline)

```
06:00  ← שלב 1: איסוף (Fetch)
       │  סריקת כל המקורות, איסוף תכנים חדשים מ-24 שעות אחרונות
       │
07:00  ← שלב 2: סינון וניקוד (Filter & Score)
       │  - ניקוד רלוונטיות לכל בן משפחה (0-100)
       │  - סינון כפילויות
       │  - סינון תוכן לא מתאים
       │
07:30  ← שלב 3: תמהיל (Curate)
       │  - בחירת TOP-N תכנים לכל בן משפחה
       │  - שילוב תוכן משפחתי
       │  - איזון בין תחומים
       │  - הוספת "גילוי" - תוכן מתחום של בן משפחה אחר
       │
08:00  ← שלב 4: יצירת תוכן (Generate)
       │  - כתיבת סיכומים בעברית/אנגלית
       │  - יצירת כותרות מושכות
       │  - הכנת תמונות ממוזערות
       │
08:30  ← שלב 5: בניית HTML (Build)
       │  - רינדור הניוזלטר כ-HTML
       │  - העלאה לשרת
       │
09:00  ← שלב 6: שליחה (Send)
       │  - שליחת קישור + כותרת לכל בן משפחה בוואטסאפ
       │
21:00  ← שלב 7: משוב (Feedback)
          - שליחת סקר מהיר
          - עיבוד תוצאות
          - עדכון מודל ההעדפות
```

## 5. מנגנון משוב

### סקר מהיר בוואטסאפ

בכל ערב (21:00) נשלחת הודעת משוב:

```
היי [שם]! 👋
איך היה הניוזלטר של היום?

1️⃣ מעולה! עוד מזה
2️⃣ טוב, אבל אפשר לשפר
3️⃣ לא ממש דיבר אליי
4️⃣ יש לי הצעה (כתוב בחזרה)

השב עם המספר 💬
```

### סוגי משוב

| סוג | מנגנון | מה נלמד |
|-----|--------|---------|
| דירוג יומי | הודעת סקר בוואטסאפ | שביעות רצון כללית |
| קליקים | Analytics על ה-HTML | אילו כתבות מעניינות |
| תגובה טקסטואלית | הודעה חופשית לבוט | הצעות ספציפיות |
| "שתף" | כפתור שיתוף בניוזלטר | תוכן שווה הפצה |
| נפח קריאה | זמן שהייה בדף | עומק עניין |

### לולאת שיפור

```
משוב → ניתוח → עדכון פרופיל → שיפור תמהיל
  │                                    │
  └────────────── שבועי ───────────────┘
```

- **יומי**: התאמת ניקוד מקורות על בסיס קליקים
- **שבועי**: ניתוח מגמות, הוספת/הסרת מקורות
- **חודשי**: סקר מעמיק יותר + דו"ח משוב למשפחה

## 6. ארכיטקטורה טכנית

### מבנה הפרויקט

```
famely-neuslettr/
├── config/
│   ├── family.json          # פרופילי בני משפחה
│   ├── sources.json         # מקורות מידע
│   └── settings.json        # הגדרות כלליות
├── src/
│   ├── main.py              # אורקסטרטור ראשי
│   ├── fetcher.py           # איסוף תכנים
│   ├── scorer.py            # ניקוד וסינון
│   ├── curator.py           # תמהיל תכנים
│   ├── generator.py         # יצירת סיכומים (Claude API)
│   ├── builder.py           # בניית HTML
│   ├── uploader.py          # העלאה לשרת
│   ├── whatsapp.py          # שליחת הודעות
│   └── feedback.py          # עיבוד משוב
├── templates/
│   └── newsletter.html      # תבנית HTML
├── data/
│   ├── feedback/            # היסטוריית משוב
│   └── cache/               # מטמון תכנים
└── tests/
```

### טכנולוגיות

| רכיב | טכנולוגיה |
|-------|----------|
| שפת תכנות | Python 3.11+ |
| יצירת תוכן | Claude API (Anthropic) |
| איסוף RSS | `feedparser` |
| Web scraping | `httpx` + `selectolax` |
| HTML Templates | Jinja2 |
| שרת סטטי | Cloudflare Pages / Vercel / S3 |
| WhatsApp API | Twilio / WhatsApp Business API |
| תזמון | Cron / Cloud Scheduler |
| Analytics | Simple pixel tracking + UTM |
| אחסון | SQLite (local) / Supabase (cloud) |

### אינטגרציית WhatsApp

