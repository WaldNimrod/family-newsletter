1 Scope — Phase 1

מה נבנה, מה לא. אפס עמימות.

Phase 1 בונה pipeline מלא מקצה לקצה: סריקת מקורות → ניקוד → תמהיל → סיכומי AI → HTML איכותי → FTP upload ל-nimrod.bio → שליחת WhatsApp (או email fallback) → סקר WhatsApp יומי → קליטת ידיעות משפחתיות.

IN SCOPE (Phase 1)

Moduleמה נבנה
M1family.json + sources.json + settings.json — config files, read-only by modules
M2RSS fetcher + web scraper + YouTube metadata fetcher. Output: NCI[]
M3Dedup + scoring + curation + Claude API summaries/headlines/greeting/puzzle + merge submissions + archive to SQLite + JSON

5 M3 · Normalizer — מפרט מלא

The brain. Dedup, score, curate, generate summaries (Claude API), merge submissions, archive. Consumes most tokens.

Pipeline (exact order)

def build_edition(ncis: list[NCI], family: FamilyConfig, settings: Settings, db: Database) -> NEO: """Main pipeline function. Steps: 1. dedup(ncis, db) → filtered NCI[] (no duplicates from last 30 days) 2. score(ncis, family) → scored NCI[] per member 3. load_submissions(db) → pending FamilySubmission[] 4. curate(scored, family, settings) → selected items per member 5. generate_content(curated, submissions, settings) → AI headlines, summaries, greeting, puzzle, bridges, edited submissions 6. build_neo(...) → complete NEO object 7. archive(neo, db) → save to SQLite + JSON 8. return neo """

Step 1: Dedup

def dedup(ncis: list[NCI], db: Database) -> list[NCI]: """Remove items whose content_hash exists in content_archive from last 30 days. SQL: SELECT content_hash FROM content_archive WHERE fetched_at > date('now', '-30 days') Filter: keep NCI where nci.content_hash NOT IN existing_hashes"""

Step 2: Score

def score(ncis: list[NCI], family: FamilyConfig) -> dict[str, list[ScoredNCI]]: """Score each NCI for each member. Returns {member_id: [ScoredNCI]}. Algorithm per (nci, member): keyword_match = count of member subtopics found in (title + raw_text + tags) normalized to 0-1 by dividing by max possible matches priority_weight = {"high": 1.0, "medium": 0.6, "low": 0.3}[matched_topic.priority] source_trust = nci.source_trust (0.0-1.0) freshness = 1.0 if age < 24h, 0.7 if < 48h, 0.5 if < 7d, 0.1 if older score = keyword_match * priority_weight * source_trust * freshness * 100 Language filter: - Shaked (language_preference="en"): only score NCI where language="en" - Others: score both, but he gets +0.1 bonus """ @dataclass class ScoredNCI: nci: NCI member_id: str score: float # 0-100 matched_topic: str # which topic matched matched_priority: str

Step 3: Load Submissions

def load_pending_submissions(db: Database, cutoff_time: str) -> list[FamilySubmission]: """SELECT * FROM family_submissions WHERE status IN ('received', 'edit_failed') AND timestamp < {cutoff_time} ORDER BY timestamp ASC"""

Step 4: Curate

def curate(scored: dict[str, list[ScoredNCI]], family: FamilyConfig, settings: Settings) -> dict[str, list[ScoredNCI]]: """Select top items per member. For each member: 1. Sort by score DESC 2. Take top N (member.max_items_per_day) 3. Enforce topic balance: no more than 60% from one topic 4. Reserve 20% of slots for discovery (from another member's top items) 5. Total across all members: max 15 unique NCIs Discovery selection: For each member, pick 1 item from another member's top-scored items that the receiving member scored LOW on (surprising cross-pollination). Generate bridge_text in step 5. """

Step 5: Generate Content (Claude API calls)

async def generate_content(curated, submissions, settings, token_tracker) -> GeneratedContent: """All Claude API calls happen here. Each call is wrapped by token_tracker. Calls (in order): 1. GREETING (1 call): Prompt: "Generate a warm Hebrew family greeting for {date}. Include a fun fact about today. 1-2 sentences. Casual, warm." Model: sonnet, max_tokens: 100 2. PUZZLE for צליל (1 call): Prompt: "Create a hard math puzzle for a very smart 13-year-old. Competition level. Include yesterday's puzzle answer: {prev_answer}. Hebrew. Format: puzzle text + (תשובה מחר!)" Model: sonnet, max_tokens: 150 3. SUMMARIES + HEADLINES (batched, 1 call per 3-4 articles): Prompt per batch: "For each article below, generate: 1. A catchy headline (max 15 words) 2. A 2-sentence summary Language: {member.language_preference} Articles: [title]: [raw_text]..." Model: sonnet, max_tokens: 200 per article 4. SUBMISSION EDITING (1 call per submission): Prompt: "You are a professional newspaper editor for a family newsletter. A family member ({name}) sent this content: Type: {message_type} Text: {content_text} [Link: {link_url}] Create: 1. An engaging headline (max 10 words, Hebrew) 2. A brief edited summary (2-3 sentences, keep the original intent) Be warm, family-friendly. This WILL be published." Model: sonnet, max_tokens: 300 5. DISCOVERY BRIDGES (1 call per bridge): Prompt: "Write a short bridge text (1 sentence, Hebrew) explaining why {from_member} would recommend this {topic} article to {to_member}. Casual, fun, family tone." Model: sonnet, max_tokens: 100 6. SURVEY QUESTION (1 call): Prompt: "Generate a varied daily survey question for a family newsletter. Today's main topics were: {topic_list}. Don't repeat yesterday's question style ({prev_question_type}). Hebrew. Short. Engaging. Should feel like a fun family conversation." Model: sonnet, max_tokens: 100 """

Step 6: Build NEO

@dataclass class NEO: date: str # YYYY-MM-DD family_name: str greeting: str # from step 5.1 family_content: list[dict] # edited submissions member_sections: list[dict] # per-member curated articles with headlines/summaries discovery: list[dict] # cross-member items with bridge text trivia: dict # {puzzle, answer, history} survey_question: str # for M5 to send at 21:00 metadata: dict # {items_fetched, items_selected, sources_scanned, # submissions_count, tokens_used: {input, output, cost_usd}, # build_duration_ms}

Step 7: Archive

def archive(neo: NEO, ncis: list[NCI], db: Database): """ 1. INSERT all fetched NCIs into content_archive (ON CONFLICT IGNORE for dedup) 2. INSERT newsletter row: date, status='ready', greeting, puzzle, metadata 3. INSERT newsletter_items: one row per (member, article) pairing 4. UPDATE family_submissions: status='published', edition_date=today 5. Save JSON: archive/editions/{date}.json = full NEO 6. Save JSON: archive/raw/{date}.json = all NCIs 7. Save HTML copy: archive/html/{date}.html (after M4 renders) """

Acceptance Criteria

  • AC-M3-01: Dedup removes items with matching content_hash from last 30 days
  • AC-M3-02: Scoring produces 0-100 score per (member, article) pair
  • AC-M3-03: Shaked gets only English articles; others get Hebrew preferred
  • AC-M3-04: Every edition has a math puzzle for צליל
  • AC-M3-05: Submissions always included regardless of score (mandatory publish)
  • AC-M3-06: Token usage logged for every Claude API call
  • AC-M3-07: NEO archived to SQLite + JSON after build
  • AC-M3-08: Claude API failure → graceful degradation to title-only mode

Acceptance Criteria

  • AC-M6-01: Webhook responds 200 within 5 seconds for any message
  • AC-M6-02: Known sender + non-survey → creates FamilySubmission with status='received'
  • AC-M6-03: Media downloaded and saved to data/submissions/{date}/
  • AC-M6-04: Confirmation reply sent within 10 seconds of receiving submission
  • AC-M6-05: Unknown sender → no reply, logged as unknown
  • AC-M6-06: Survey response saved to feedback table with type='survey'
  • AC-M6-07: Media over size limit → friendly rejection message, no DB entry
